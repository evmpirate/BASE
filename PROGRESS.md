## Design decisions

- **agentURI is a `data:application/json;base64` URI** — the registration file (agent card)
  lives fully on-chain, consistent with the badges collection; no dead links, no hosting
  dependency. Generated by `make-card.mjs`. If a stable public URL appears later (e.g. Vercel),
  swap it with `setAgentURI(8073, newUri)`.
- Two-step registration: `register(cardV1)` -> read agentId from Transfer event ->
  `setAgentURI` with the final card containing the `registrations` back-reference
  `{ agentId: 8073, agentRegistry: "eip155:84532:0x8004A818..." }`.
- Registered from the user's main wallet so the agent NFT is owned by the holder of
  dupcia.base.eth (the basename itself lives on Base mainnet; the card lists it under `services`).
- Registry addresses taken from the ERC-8004 reference deployment (github.com/erc-8004/erc-8004-contracts),
  verified on-chain (proxy + implementation ABI fetched from BaseScan before calling).

## Service (`server.js`, Node + express + viem)

- `GET /.well-known/agent-card.json` — agent card incl. registration binding (AGENT_ID=8073 env)
- `GET /progress` — live read of OnchainTrailBadges: badge list + milestone journey status
- Run: `AGENT_ID=8073 node server.js` (port 4021)
